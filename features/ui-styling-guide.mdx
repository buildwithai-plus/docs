## Why NativeWind vs. StyleSheets

### Traditional StyleSheet Approach

```tsx
import { StyleSheet, View, Text } from 'react-native';

function TraditionalComponent() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello World</Text>
      <Text style={styles.subtitle}>Welcome to my app</Text>
      <View style={styles.button}>
        <Text style={styles.buttonText}>Click me</Text>
      </View>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    padding: 16,
    backgroundColor: '#f0f0f0',
    borderRadius: 8,
  },
  title: {
    fontSize: 18,
    fontWeight: 'bold',
    marginBottom: 8,
    color: '#333333',
  },
  subtitle: {
    fontSize: 14,
    color: '#666666',
  },
  button: {
    marginTop: 16,
    padding: 12,
    backgroundColor: '#5850EC',
    borderRadius: 4,
    alignItems: 'center',
  },
  buttonText: {
    color: '#ffffff',
    fontWeight: '500',
  },
});
```

### NativeWind Approach

```tsx
import { View } from 'react-native';
import { Text } from '~/components/ui/text';

function NativeWindComponent() {
  return (
    <View className="p-4 bg-muted rounded-lg">
      <Text className="text-lg font-bold mb-2 text-foreground">Hello World</Text>
      <Text className="text-sm text-muted-foreground">Welcome to my app</Text>
      <View className="mt-4 p-3 bg-primary rounded items-center">
        <Text className="text-primary-foreground font-medium">Click me</Text>
      </View>
    </View>
  );
}
```

### Benefits of NativeWind

1. **Readability** - Styles are right where you use them, making components easier to understand
2. **No naming headaches** - No need to come up with unique style names for every component
3. **Theme support** - Use theme variables like `bg-primary` that automatically adapt to light/dark modes
4. **Consistency** - Predefined spacing, colors, and typography scales keep your UI consistent
5. **Less code** - What takes 30+ lines with StyleSheet can often be done in 10 lines with NativeWind
6. **No style merging** - Avoid complex StyleSheet merging logic for conditional styling

## Getting Started with NativeWind

NativeWind lets you style components using simple class names instead of complex style objects. It works just like Tailwind CSS but for React Native.

### Basic NativeWind Classes

```tsx
// Import UI components from library and View from react-native
import { View } from "react-native";
import { Text } from "~/components/ui/text";

// Add padding (p-4 = padding of size 4)
<View className="p-4">
  {/* Make text larger (text-lg) and bold */}
  <Text className="text-lg font-bold">Hello World</Text>
</View>

// Put items side by side (flex-row) with space between (gap-2)
<View className="flex-row items-center gap-2">
  <Text>ðŸ‘‹</Text>
  <Text>Welcome!</Text>
</View>
```

### Common NativeWind Classes Explained

**Layout:**
- `p-2`, `p-4`, `p-6` - Padding (space inside) small, medium, large
- `m-2`, `m-4`, `m-6` - Margin (space outside) small, medium, large
- `flex-row` - Position items horizontally (side by side)
- `flex-col` - Position items vertically (default)
- `items-center` - Center items vertically when in row, horizontally when in column
- `justify-center` - Center items horizontally when in row, vertically when in column
- `gap-2`, `gap-4` - Space between items

**Typography:**
- `text-sm`, `text-base`, `text-lg`, `text-xl` - Text sizes
- `font-bold`, `font-medium` - Text weights
- `text-center`, `text-left` - Text alignment

**Colors:**
- `bg-primary` - Background using your theme's primary color
- `text-primary` - Text using your theme's primary color
- `text-primary-foreground` - Text color designed to be visible on primary background
- `border-primary` - Border using your theme's primary color

**Shapes:**
- `rounded-md`, `rounded-lg`, `rounded-xl` - Rounded corners (small, medium, large)
- `border-2` - Border with thickness 2
- `w-full` - Full width
- `h-12`, `h-24` - Fixed height

## React Native Reusables Component Library

Your app includes pre-built components from [React Native Reusables](https://www.reactnativereusables.com/) - shadcn/ui for React Native. These components:

- Work across **iOS, Android, and web** platforms
- Support **light and dark mode** automatically
- Are built with **accessibility** in mind
- Can be styled with NativeWind classes

### Component Features

- **Variants** - Pre-defined styles like `variant="outline"` or `size="sm"`
- **className** - Use NativeWind classes to customize further
- **Props** - Additional customization through standard React props

### Key Components

- **Button** - Different variants, sizes, and states
- **Text** - Typography with semantic variants
- **Card** - Container with header, content, and footer sections
- **Input** - Text fields with validation states
- **Avatar** - User profile images with fallbacks

### Adding More Components

Add more components with the CLI:

```bash
npx @react-native-reusables/cli@latest add component-name
```

Example: If you want to add a progress bar component, you can run in the terminal:

```bash
npx @react-native-reusables/cli@latest add progress-bar
```

You can find all available components [here](https://www.reactnativereusables.com/components).

## Complete Example

Here's a complete example using Card, Button, and Text components:

```tsx
import { View } from "react-native";
import { Text } from "~/components/ui/text";
import { Button } from "~/components/ui/button";
import { Card, CardContent, CardHeader, CardFooter } from "~/components/ui/card";

function ProductCard() {
  return (
    <Card className="m-4 overflow-hidden border border-border">
      {/* CardHeader with padding (p-4) and centered content */}
      <CardHeader className="p-4 bg-muted">
        <Text className="text-lg font-bold text-center">Premium Plan</Text>
      </CardHeader>

      {/* CardContent with padding and space between items */}
      <CardContent className="p-4 gap-4">
        <Text className="text-secondary-foreground">All features including:</Text>
        
        {/* Feature items with icon and text */}
        <View className="flex-row items-center gap-2">
          <Text>âœ“</Text>
          <Text>Unlimited projects</Text>
        </View>
        
        <View className="flex-row items-center gap-2">
          <Text>âœ“</Text>
          <Text>Priority support</Text>
        </View>

        {/* Price with large text */}
        <Text className="text-xl font-bold text-center mt-2">$9.99/month</Text>
      </CardContent>

      {/* CardFooter with padding and centered button */}
      <CardFooter className="p-4 bg-background border-t border-border">
        {/* Primary button with full width */}
        <Button 
          className="w-full" 
          variant="default">
          {/* Always wrap text in Button with Text component */}
          <Text className="text-primary-foreground font-medium">Subscribe Now</Text>
        </Button>
      </CardFooter>
    </Card>
  );

```

### What This Example Shows

1. **Imports** - Get UI components from `~/components/ui/` and basic components like `View` from React Native
2. **Card Layout** - Header, Content, and Footer sections
3. **NativeWind Classes** - Using spacing (`p-4`, `gap-4`), colors (`bg-muted`), and typography (`text-lg`)
4. **Button Usage** - Always wrap text inside Button with the `<Text>` component
5. **Layout Structure** - Flex row for side-by-side items, centering with `text-center`

## Quick Tips

1. **Combine approaches** - Use component variants for consistent styling, then customize with className when needed
2. **Check dark mode** - Test your UI in both light and dark themes to ensure good contrast
3. **Learn from examples** - Study the pre-built components to see best practices
4. **Use theme colors** - Stick with theme variables like `bg-primary` instead of hardcoded colors
5. **Ask the AI** - Need help with styling? Just ask!

## Adding Icons

Your app comes with many icons ready to use. Need more? Here's how:

**Example: Adding a Heart icon**

1. **Create** `~/lib/icons/Heart.tsx`:
```tsx
import { Heart } from 'lucide-react-native';
import { iconWithClassName } from './iconWithClassName';

export { Heart };
```

2. **Use it anywhere:**
```tsx
import { Heart } from "~/lib/icons/Heart"

<Heart className="text-primary" size={24} />
```
