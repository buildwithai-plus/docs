## Troubleshooting

### Common Issues and Solutions

#### "My colors don't change in dark mode"

**Problem:** You're using hardcoded colors instead of theme variables.

**Solution:** Replace hardcoded colors with theme color classes:

```tsx
// ❌ Don't do this
<View style={{ backgroundColor: '#3B82F6' }}>
  <Text style={{ color: 'white' }}>Button</Text>
</View>

// ✅ Do this instead
<View className="bg-primary">
  <Text className="text-primary-foreground">Button</Text>
</View>
```

#### "Component not found error"

**Problem:** You haven't installed the component or you're importing it incorrectly.

**Solution:** Install the component and use the correct import path:

```tsx
// First, install the component
// npx @react-native-reusables/cli@latest add button

// ❌ Don't do this
import { Button } from 'react-native-reusables';

// ✅ Do this instead
import { Button } from '~/components/ui/button';
```

#### "Styles look different on iOS vs Android"

**Problem:** Some styles have platform-specific differences.

**Solution:** Use platform-specific styles when needed:

```tsx
// Use conditional classes for platform differences
<View className={Platform.OS === 'ios' ? 'shadow-lg' : 'elevation-3'}>
  {/* Content */}
</View>
```

#### "How to override component styles"

**Problem:** You want to customize a component's appearance.

**Solution:** Use the `className` prop or modify the component source:

```tsx
// Option 1: Use className prop
<Button className="rounded-full px-8">
  <Text>Custom Button</Text>
</Button>

// Option 2: Modify the component source in components/ui/button.tsx
```

#### "My flex layout isn't working as expected"

**Problem:** Flex layouts can be tricky, especially with nested components.

**Solution:** Make sure parent containers have the right flex settings:

```tsx
// Make sure parent has flex-1 if you want children to expand
<View className="flex-1">
  <View className="flex-1 bg-red-500" />
  <View className="flex-2 bg-blue-500" />
</View>
```

#### "Text doesn't wrap correctly"

**Problem:** Text overflows its container.

**Solution:** Use text wrapping properties:

```tsx
<Text className="flex-wrap">
  This is a long text that should wrap to multiple lines when it reaches the edge of its container.
</Text>
```

#### "Shadows don't appear on Android"

**Problem:** The `shadow` class works differently on Android.

**Solution:** Use elevation for Android:

```tsx
<View className={Platform.OS === 'ios' ? 'shadow-lg' : 'android:elevation-5'}>
  {/* Content */}
</View>
```

## Best Practices

1. **Use theme colors** for consistency and dark mode support
2. **Compose small components** into larger ones for maintainability
3. **Keep className strings readable** by breaking them into logical groups
4. **Use semantic color names** (`primary`, `destructive`) instead of visual names (`blue`, `red`)
5. **Customize components at the source** when making app-wide changes
6. **Use consistent spacing** with the same padding/margin values throughout your app
7. **Test in both light and dark mode** to ensure good contrast
8. **Use React Native Reusables** for complex components instead of building from scratch
