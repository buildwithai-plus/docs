# Customizing Colors

One of the most powerful features of the theme system is how easy it makes customizing your app's colors. In this section, you'll learn how to modify your theme colors to match your brand while maintaining light/dark mode support.

## What You'll Learn

- How to locate and modify your theme colors
- How to work with HSL color values
- How to ensure colors work in both light and dark mode
- How to test your color changes

## Finding Your Theme Colors

Your app's theme colors are defined in a file called `global.css`. This file contains all the color variables that power your app's design system.

To locate this file:

1. Open your project in your code editor
2. Navigate to the `src` directory
3. Look for the `global.css` file (usually at `src/global.css`)

When you open this file, you'll see something like this:

```css
/* Light mode colors */
:root {
  --background: 0 0% 100%;      /* White background */
  --foreground: 222 47% 11%;    /* Dark text */
  --primary: 221 83% 53%;       /* Blue */
  --primary-foreground: 0 0% 100%;  /* White text on primary */
  --card: 0 0% 100%;            /* White card background */
  --card-foreground: 222 47% 11%;   /* Dark text on card */
  --muted: 210 40% 96%;         /* Light gray background */
  --muted-foreground: 215 16% 47%;  /* Medium gray text */
  /* ... other colors ... */
}

/* Dark mode colors */
.dark:root {
  --background: 222 47% 11%;    /* Dark background */
  --foreground: 210 40% 98%;    /* Light text */
  --primary: 217 91% 60%;       /* Brighter blue */
  --primary-foreground: 0 0% 100%;  /* White text on primary */
  --card: 222 47% 11%;          /* Dark card background */
  --card-foreground: 210 40% 98%;   /* Light text on card */
  --muted: 217 33% 17%;         /* Dark gray background */
  --muted-foreground: 215 20% 65%;  /* Light gray text */
  /* ... other colors ... */
}
```

## Understanding HSL Color Format

Your theme uses HSL (Hue, Saturation, Lightness) color values, which make it easier to create harmonious color schemes.

The format is: `H S% L%` where:
- **H (Hue)**: 0-360, represents the color on the color wheel (0=red, 120=green, 240=blue)
- **S (Saturation)**: 0-100%, how vivid the color is (0%=gray, 100%=full color)
- **L (Lightness)**: 0-100%, how light the color is (0%=black, 100%=white)

## Changing Your Primary Color

Let's walk through changing your app's primary color from blue to purple:

### Before:

```css
/* Light mode */
:root {
  --primary: 221 83% 53%;       /* Blue */
}

/* Dark mode */
.dark:root {
  --primary: 217 91% 60%;       /* Brighter blue */
}
```

### After:

```css
/* Light mode */
:root {
  --primary: 280 75% 60%;       /* Purple */
}

/* Dark mode */
.dark:root {
  --primary: 280 85% 70%;       /* Brighter purple */
}
```

### What Changed:

1. We changed the **hue** from `221` (blue) to `280` (purple)
2. We adjusted the **saturation** and **lightness** for both light and dark mode
3. For dark mode, we made the purple brighter (higher lightness) to ensure it stands out

## Creating a Complete Color Scheme

When changing colors, it's important to maintain proper contrast and consistency. Here's how to create a complete color scheme:

1. **Choose your primary color** - This is your brand's main color
2. **Create a darker version** for hover states (reduce lightness by 10%)
3. **Create a lighter version** for backgrounds (reduce saturation, increase lightness)
4. **Ensure text contrast** by testing light and dark text on your colors

### Example: Creating a Green Theme

```css
/* Light mode - Green theme */
:root {
  --background: 0 0% 100%;      /* White background */
  --foreground: 222 47% 11%;    /* Dark text */
  
  --primary: 142 70% 45%;       /* Green */
  --primary-foreground: 0 0% 100%;  /* White text on green */
  
  --card: 0 0% 100%;            /* White card background */
  --card-foreground: 222 47% 11%;   /* Dark text on card */
  
  --muted: 142 33% 96%;         /* Light green background */
  --muted-foreground: 142 30% 30%;  /* Dark green text */
}

/* Dark mode - Green theme */
.dark:root {
  --background: 222 47% 11%;    /* Dark background */
  --foreground: 210 40% 98%;    /* Light text */
  
  --primary: 142 70% 50%;       /* Brighter green */
  --primary-foreground: 0 0% 100%;  /* White text on green */
  
  --card: 222 47% 11%;          /* Dark card background */
  --card-foreground: 210 40% 98%;   /* Light text on card */
  
  --muted: 142 33% 20%;         /* Dark green background */
  --muted-foreground: 142 30% 80%;  /* Light green text */
}
```

## Testing Your Colors

After changing your theme colors, it's important to test them in both light and dark mode:

1. **Light mode testing**: View your app in normal mode to ensure colors look good
2. **Dark mode testing**: Toggle dark mode to verify colors have proper contrast
3. **Component testing**: Check all components that use your primary color

### Common Color Issues to Watch For:

- **Low contrast**: Text that's hard to read against its background
- **Clashing colors**: Colors that don't work well together
- **Inconsistent feel**: Light/dark mode versions that feel like different brands

## How Light/Dark Mode Works

Your theme automatically switches between light and dark mode colors based on the user's system preferences. Here's how it works:

1. The system detects if the user has dark mode enabled
2. If dark mode is on, the `.dark:root` CSS variables are applied
3. If dark mode is off, the `:root` CSS variables are used
4. All components using theme colors automatically update

You don't need to write any conditional code in your components - just use theme color classes like `bg-primary` and they'll automatically adapt!
