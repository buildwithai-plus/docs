## Setup Process

<Steps>
  <Step title="Clone the Boilerplate">
    <Tabs>
      <Tab title="Cursor IDE (Recommended)">
        1. Open **Cursor IDE**
        2. Click **Clone Repo** (or press `Ctrl+Shift+P` and search "Git: Clone")
        3. Paste this GitHub repository URL:
           ```
           https://github.com/steffensunbranch/buildwithai-boilerplate.git
           ```
        4. Click **Clone from GitHub**
        5. Sign in to GitHub when prompted
        6. Choose where to save your project
        7. Click **Open** when the clone completes
      </Tab>
      <Tab title="Windsurf IDE">
        1. Open **Windsurf IDE**
        2. Click **Clone Repo** or use Command Palette (`Ctrl+Shift+P`)
        3. Paste this GitHub repository URL:
           ```
           https://github.com/steffensunbranch/buildwithai-boilerplate.git
           ```
        4. Click **Clone from GitHub**
        5. Sign in to GitHub when prompted
        6. Choose where to save your project
        7. Click **Open** when the clone completes
      </Tab>
    </Tabs>

    <Info>
      This method automatically handles the Git setup and opens your project in the correct location.
    </Info>
  </Step>
  <Step title="Install Dependencies">
    Open the integrated terminal in your IDE (`Ctrl+J` / `Cmd+J` for Cursor, or `Ctrl+Shift+` for Windsurf) and run:

    ```bash
    npm install
    ```

    <Info>
      This will download and install all required packages. It may take several minutes to complete.
    </Info>
  </Step>
  <Step title="Set Up Environment Variables">
    Create your environment file by copying the example:

    ```bash
    cp .env.example .env
    ```

    <Accordion title="What are environment variables?">
      Environment variables store configuration settings that may change between environments (development, staging, production).

      Your .env file will store API keys, service URLs, and other configuration values that should not be committed to source control.

      You'll add specific values as you follow the next guides in this documentation.
    </Accordion>
  </Step>
</Steps>
