## Configure Your App

After setting up your project repository and Supabase authentication, the next step is to configure your app's essential settings before creating a development build.

### Step 1: Configure App Settings

The boilerplate comes with an example configuration file. Start by renaming it to `app.json`:

```bash
cp app.json.example app.json
```

Now, open the `app.json` file and update the following essential fields to customize your app:

```json
{
  "expo": {
    "name": "Your App Name",
    "slug": "your-app-slug",
    "scheme": "your-app-scheme",
    "ios": {
      "bundleIdentifier": "com.yourcompany.yourapp"
    },
    "android": {
      "package": "com.yourcompany.yourapp"
    }
  }
}
```

<Tip>
  The boilerplate includes many more configuration options, but these are the essential fields you need to update before starting development.
</Tip>

#### Essential fields to update:

You'll need to modify these settings in your project's `app.json` file:

<CardGroup cols={1}>
  <Card title="name" icon="mobile-screen">
    **What it is**: The display name of your app as it will appear on users' devices
    
    **Example**: `FitTracker Pro` or `Recipe Finder`
    
    **Tip**: Choose something descriptive and memorable
  </Card>

  <Card title="slug" icon="link">
    **What it is**: A unique identifier for your project used in URLs and directories
    
    **Example**: `fittracker-pro` or `recipe-finder`
    
    **Rules**: Use lowercase letters, hyphens instead of spaces, no special characters
    
    **⚠️ WARNING**: The slug **cannot be changed** after publishing without creating a new project!
  </Card>

  <Card title="scheme" icon="arrow-right-to-bracket">
    **What it is**: A URL scheme for deep linking to your app from other apps or websites
    
    **Example**: `fittrackerapp` or `recipefinderapp`
    
    **Tip**: Choose something unique with no spaces or special characters
  </Card>

  <Card title="bundleIdentifier (iOS)" icon="apple">
    **What it is**: Your app's unique identifier in Apple's ecosystem
    
    **Example for individuals**: `io.github.username.appname` or `dev.yourname.appname`
    
    **Format**: Use reverse domain notation - for individuals without a company, use your GitHub username or personal domain
  </Card>

  <Card title="package (Android)" icon="android">
    **What it is**: Your app's unique identifier in Google's ecosystem
    
    **Example for individuals**: `io.github.username.appname` or `dev.yourname.appname`
    
    **Tip**: Should match your iOS bundleIdentifier for consistency
  </Card>
</CardGroup>

### Step 3: Set Up Environment Variables

Create your environment file from the example:

```bash
cp .env.example .env
```

Update the variables in `.env` with your configuration:

```env
# App Environment
APP_ENV=development

# Supabase Configuration
# (You'll get these after setting up Supabase)
EXPO_PUBLIC_SUPABASE_URL=
EXPO_PUBLIC_SUPABASE_ANON_KEY=

# Optional: Analytics Configuration
EXPO_PUBLIC_GOOGLE_ANALYTICS_ID=
```
