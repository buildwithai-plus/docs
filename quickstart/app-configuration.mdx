# Configure Your App

After setting up your project, the next step is to configure your app's essential settings and visual identity before creating a development build.

### Step 1: Configure App Settings

The boilerplate comes with an example configuration file. Copy and paste this command into your terminal and press Enter to create your app.json file:

```bash
cp app.json.example app.json
```

### Step 2: Configure App Icons & Splash Screens

Customizing your app's icons and splash screens helps create a professional, branded experience for your users.

#### Current Default Assets

Your app currently uses these default placeholder assets:

![Default App Icon and Splash Screen](../assets/images/default-app-assets.png)
*Default icon and splash screen that come with the boilerplate*

#### Create Your Custom Assets

**Need help creating icons?** Use these free tools to generate app icons in the right sizes:
- [Canva App Icon Maker](https://www.canva.com/create/app-icons/) - Free templates and easy resizing
- [App Icon Generator](https://appicon.co/) - Upload one image, get all sizes
- [IconKitchen](https://icon.kitchen/) - Free icon generator with templates

##### Asset Requirements

**App Icons:**
- **Size**: 1024×1024 pixels
- **Format**: PNG with transparency
- **Style**: Simple, recognizable design that works at small sizes

**Splash Screen:**
- **Size**: At least 1242×2436 pixels (iPhone X resolution)
- **Content**: Your app logo centered on a solid background
- **Format**: PNG or JPG

#### Add Your Assets

1. **Locate the images folder**: In your project root, find the `assets/images` folder
2. **Replace the default files**:
   - Replace `assets/images/icon.png` with your app icon
   - Replace `assets/images/splash.png` with your splash screen

#### Configuration

In your `app.json`, update the following sections (highlighted in blue):

```json
{
  "expo": {
    "icon": "./assets/images/icon.png",
    "splash": {
      "image": "./assets/images/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    }
  }
}
```

<Info>
  The boilerplate includes placeholder images that you can use as templates for sizing and positioning.
</Info>

### Step 3: Configure App Settings

#### Essential fields to update:

You'll need to modify these settings in your project's `app.json` file:

- **name**: The display name shown on users' devices (e.g., `Travel App`)  
  *Choose something descriptive and memorable*

- **slug**: Unique identifier used in URLs and directories (e.g., `travel-app`)  
  *Use lowercase letters and hyphens only*  
  **⚠️ Warning:** Cannot be changed after publishing!

- **scheme**: URL scheme for deep linking (e.g., `travelapp`)  
  *No spaces or special characters*

- **bundleIdentifier** (iOS): Unique identifier in Apple's ecosystem (e.g., `com.buildwithai.travelapp`)  
  *Use reverse domain notation where "owner" can be your name, domain, or company*

- **package** (Android): Unique identifier in Google's ecosystem (e.g., `com.buildwithai.travelapp`)  
  *Should match your iOS bundleIdentifier for consistency*

#### Update Your app.json

Open the `app.json` file and update the following essential fields (change the <span style="color: #0066cc; font-weight: bold;">blue text</span> to your own values):

```json
{
  "expo": {
    "name": "Your App Name",
    "slug": "your-app-slug", 
    "scheme": "your-app-scheme",
    "ios": {
      "bundleIdentifier": "com.owner.yourapp"
    },
    "android": {
      "package": "com.owner.yourapp"
    }
  }
}
```

<Tip>
  The boilerplate includes many more configuration options, but these are the essential fields you need to update before starting development.
</Tip>
