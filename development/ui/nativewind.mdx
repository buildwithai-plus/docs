## NativeWind Theming

The BuildWithAI+ boilerplate uses [NativeWind](https://www.nativewind.dev/) as its styling solution, which brings the power and convenience of Tailwind CSS to React Native. This allows you to create beautiful, responsive UIs using the familiar Tailwind class-based approach.

### How Theming Works in the Boilerplate

The theming system in the boilerplate is built on several key components:

1. **TailwindCSS Configuration**: A custom `tailwind.config.js` file that defines your theme's colors, spacing, typography, and more.

2. **Theme Provider**: A context provider that manages theme state (light/dark mode) and provides theme-related utilities throughout the app.

3. **Dynamic Styling**: Utilities for switching between light and dark mode and handling responsive layouts.

### Theme Structure

The theme configuration is located in `tailwind.config.js` and is structured like this:

```javascript
module.exports = {
  content: ["./src/**/*.{js,jsx,ts,tsx}"],
  theme: {
    extend: {
      colors: {
        primary: {
          50: '#e6f6ff',
          100: '#b3e0ff',
          200: '#80cbff',
          300: '#4db5ff',
          400: '#1a9fff',
          500: '#0080ff',  // Base primary color
          600: '#0066cc',
          700: '#004d99',
          800: '#003366',
          900: '#001a33',
        },
        secondary: {
          // Secondary color palette...
        },
        // Additional color palettes...
      },
      // Other theme extensions...
    },
  },
  plugins: [],
};
```

### Using NativeWind in Your Components

#### Basic Usage

To apply styles to a component, simply use the `className` prop:

```jsx
import { Text, View } from 'react-native';

export function MyComponent() {
  return (
    <View className="p-4 bg-white dark:bg-gray-900">
      <Text className="text-lg font-bold text-gray-800 dark:text-white">
        Hello World
      </Text>
    </View>
  );
}
```

#### Conditional Styling

You can conditionally apply styles using JavaScript expressions:

```jsx
<View
  className={`p-4 rounded-lg ${
    isActive ? 'bg-primary-500' : 'bg-gray-200'
  }`}
>
  <Text className="text-white">
    {isActive ? 'Active' : 'Inactive'}
  </Text>
</View>
```

#### Responsive Design

NativeWind supports responsive design through breakpoints:

```jsx
<View className="flex-col md:flex-row">
  {/* Will be column on small screens, row on medium screens and up */}
</View>
```

The boilerplate defines these screen size breakpoints:
- `sm`: 640px
- `md`: 768px
- `lg`: 1024px
- `xl`: 1280px
- `2xl`: 1536px

### Customizing the Theme

#### Modifying Colors

To change the color palette of your app:

1. Open `tailwind.config.js`
2. Update the color values in the `theme.extend.colors` section:

```javascript
colors: {
  primary: {
    50: '#f0f9ff',  // Lightest shade
    // ...other shades
    500: '#0284c7',  // Base color (change this to your brand color)
    // ...other shades
    900: '#082f49',  // Darkest shade
  },
  // Add or modify other color palettes
}
```

#### Adding Custom Fonts

To add custom fonts to your theme:

1. Add your font files to the `assets/fonts` directory
2. Register them in `app.json` under the `expo.fonts` section
3. Update the `fontFamily` section in `tailwind.config.js`:

```javascript
fontFamily: {
  'sans': ['Inter-Regular'],
  'sans-medium': ['Inter-Medium'],
  'sans-bold': ['Inter-Bold'],
  // Add your custom fonts here
}
```

4. Use them in your components:

```jsx
<Text className="font-sans-bold">Bold Text</Text>
```

### Dark Mode Support

The boilerplate includes full support for dark mode:

1. **Automatic Theme Detection**: The app detects the user's system theme preference

2. **Manual Theme Switching**: Users can override the system preference

3. **Styling for Dark Mode**: Use `dark:` prefix to apply dark mode styles:

```jsx
<View className="bg-white dark:bg-gray-900">
  <Text className="text-black dark:text-white">
    This text adapts to theme
  </Text>
</View>
```

### Theme Context

The boilerplate provides a theme context that you can use to access and modify the current theme:

```jsx
import { useTheme } from '@/providers/ThemeProvider';

function MyComponent() {
  const { theme, toggleTheme, isDarkMode } = useTheme();
  
  return (
    <View>
      <Text>Current theme: {theme}</Text>
      <Button onPress={toggleTheme} title="Toggle Theme" />
    </View>
  );
}
```

### Best Practices

1. **Use Semantic Color Names**: When extending the theme, use semantic names for colors rather than descriptive ones (`primary`, `error` instead of `blue`, `red`)

2. **Maintain Color Scales**: For each color, maintain a full scale from 50 to 900 for consistency

3. **Group Related Styles**: Use Tailwind's grouping capabilities to keep your styles organized:

```jsx
<View className="flex-row items-center justify-between">
  {/* This is clearer than listing all properties separately */}
</View>
```

4. **Extract Common Patterns**: For repeated style patterns, create reusable components
