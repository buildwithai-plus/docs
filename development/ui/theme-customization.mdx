## Overview

You can customize your app's theme colors using the Tweakcn Editor - a visual tool that generates NativeWind (TailwindCSS) compatible color schemes.

<Frame caption="Tweakcn Editor Interface - Visual Theme Designer">
  ![Tweakcn Editor showing the theme customization interface](/images/theme-editor.png)
</Frame>

## Prerequisites

Before customizing your theme, ensure you have:
- A running development build of your app
- Access to your project's `global.css` file
- Access to your project's `lib/constants.ts` file

## Designing Your Theme

<Steps>
  <Step title="Open the Theme Editor">
    Navigate to the [Tweakcn Editor](https://tweakcn.com/editor/theme?tab=ai) in your browser.
  </Step>

  <Step title="Create Your Color Scheme">
    1. Use the AI chat to describe your desired theme, or
    2. Manually adjust colors using the visual controls
    3. Test your theme by toggling between light and dark modes using the switch in the top middle

    <Tip>
      Preview your theme in both light and dark modes to ensure sufficient contrast and readability.
    </Tip>
  </Step>

  <Step title="Export Theme Colors">
    1. Click the `{ } Code` button in the top right
    2. In the dropdown menus, select:
       - Format: **Tailwind v4**
       - Color Format: **HSL**
    3. Click **Copy** to get your theme code

    <Frame caption="Theme Code Export Interface">
      ![Theme code export dialog showing format options](/images/theme-code.png)
    </Frame>
  </Step>
</Steps>

## Applying Your Theme

<Steps>
  <Step title="Update Global CSS">
    Copy the entire theme code from Tweakcn into your `global.css` file, but:

    1. Keep these three imports at the top of your file:
    ```css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    ```

    2. Change `.dark` to `.dark:root` in the selectors

    For reference, here's how your current `global.css` looks:
    ```css
    @layer base {
      :root {
        --background: 0 0% 100%;
        --foreground: 222.2 84% 4.9%;
        --card: 0 0% 100%;
        --card-foreground: 222.2 84% 4.9%;
        --destructive: 0 84.2% 60.2%;
        --primary: 221.2 83.2% 53.3%;
        /* ... and so on */
      }

      .dark:root {
        --background: 222.2 84% 4.9%;
        --foreground: 210 40% 98%;
        /* ... dark mode colors */
      }
    }
    ```

    <Info>
      The new theme code will have the same structure and variable names, so you can safely replace everything after the imports.
    </Info>
  </Step>

  <Step title="Update Color Constants">
    Update the color arrays in `lib/constants.ts`:

    ```typescript
    export const NAV_THEME = {
      light: {
        background: 'hsl(0 0% 100%)',
        text: 'hsl(222.2 84% 4.9%)',
        primary: 'hsl(221.2 83.2% 53.3%)',
        // ... other light mode colors
      },
      dark: {
        background: 'hsl(222.2 84% 4.9%)',
        text: 'hsl(210 40% 98%)',
        // ... dark mode colors
      }
    };
    ```

    <Info>
      Theme colors are organized in light/dark modes using HSL format for consistent color management.
    </Info>
  </Step>

  <Step title="Update App Configuration">
    If needed, update splash screen colors in `app.json`:

    ```json
    {
      "expo": {
        "splash": {
          "backgroundColor": "#ffffff" // Use HEX format here
        }
      }
    }
    ```
  </Step>
</Steps>

## Verifying Your Changes

<Steps>
  <Step title="Restart Development Server">
    Restart your development server to apply the theme changes.
  </Step>

  <Step title="Test Theme Changes">
    1. Check all UI components in light mode
    2. Toggle dark mode and verify components
    3. Ensure text remains readable in both modes
    4. Test on physical devices for accurate color representation
  </Step>
</Steps>

<Note>
  Colors may appear slightly different on physical devices compared to simulators. Always test your theme on actual devices before deploying.
</Note>
